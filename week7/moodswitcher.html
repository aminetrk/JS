Questions

1. Why is the JavaScript textContent property safer than innerHTML?

textContent is safer because it only adds plain text. innerHTML can run HTML or scripts, which can be unsafe. textContent protects the page from bad code.

2. How do the id="text" attribute and the DOM property relate?

The id="text" is the name of the element in HTML. When we use getElementById("text"), JavaScript finds that same element, so the variable and the HTML element are connected.

3. What happens if we run text.textContent = "Hello!" ?

The text inside the element with id="text" will change to the word "Hello!".
